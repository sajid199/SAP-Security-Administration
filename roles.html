üìå Purpose

PFCG is used to create, change, and assign roles that control access and navigation in SAP.

üß© Tabs in PFCG Role

1. Description Tab

  • Used to define the role name and description.
  • Optional: Target system in CUA environment.

2. Menu Tab

  • Defines navigation structure for end-users.
  • Contains:
    • Transaction codes (T-Codes)
    • Reports
    • Web links
  • Note: Adding a T-Code here does NOT grant access until authorized.

3. Workflow Tab

  • Used to link roles with workflows.
  • Enables task routing via SAP Business Workplace (SBWP).

4. Authorizations Tab

  • The most important tab ‚Äì controls what a user can do.
  • Contains:
    • Authorization objects (like S_TCODE)
    • Activities (read, create, delete, etc.)
  • After maintaining, you must generate the profile.

5. User Tab

  • Assign users to the role.
  • Set start and end date for validity.

6. MiniApps Tab

  • Used in SAP Enterprise Portal to assign small application tiles (MiniApps).
  • Not commonly used in standard GUI roles.

7. Personalization Tab

  • Allows setting user-specific parameters.
  • Example: Date format, default printer, etc.
